# Code Plan

`code_plan/` 是这个项目的人工维护快照目录，用来让新对话在没有上下文记忆时，快速了解：

- 项目当前在做什么
- 已经实现了哪些功能
- 哪些功能还不稳定或待完善
- 哪些方向建议继续推进
- 哪些方向当前建议忽略
- 我们希望后续协作时遵守什么更新习惯

它不是 Git 的替代品，但可以看作“面向对话接管”的轻量项目状态层。

## 新对话建议读取顺序

1. `PROJECT_SNAPSHOT.md`
2. `FEATURE_STATUS.md`
3. `ROADMAP_NOTES.md`
4. `WORKING_PROTOCOL.md`

## 维护原则

- 这里记录的是“当前工作树状态”，不保证等于远端 `main`。
- 只记录结构性信息、稳定结论、关键决策、主要风险。
- 不记录琐碎临时输出，不替代 Git commit history。
- 发生以下情况时，建议同步更新 `code_plan/`：
  - 新增或重构核心功能
  - 数据流、目录结构、部署方式发生变化
  - 某个外部依赖或数据源状态出现实质变化
  - 新增明确“不建议做”的路线
  - 前端筛选维度、导出接口、数据 schema 有结构调整

## 当前目录文件说明

- `PROJECT_SNAPSHOT.md`
  - 当前项目目标、运行方式、数据来源、现状摘要
- `FEATURE_STATUS.md`
  - 功能清单、完成状态、已知问题
- `ROADMAP_NOTES.md`
  - 建议增加功能、建议忽略功能、近期优先级
- `WORKING_PROTOCOL.md`
  - 新对话如何读取、如何更新、何时提醒维护 `code_plan/`

## 对后续助手的默认要求

- 新对话开始时，如果任务涉及结构改动、部署、抓取链路、前端筛选、schema 变更，优先读取本目录。
- 如果本目录内容与代码明显不一致，以代码为准，并建议更新本目录。
- 如果本次修改属于结构性优化，完成后应主动判断是否要补写或更新 `code_plan/`。
